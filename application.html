兩百字內的個人簡介

在2013~2017年中是童書編輯，在近幾年臺灣的電子書領域開始重新發展的時候，乘著好奇的心態，也想試試看能不能與這個產業一起成長，於是在2017年中轉職成為電子書編輯

在職涯裡，覺得自己是：在工作過程中，從每次學到的東西，不停延伸發現未來有哪些可能的人，並且享受其中。（很饒口哈哈）

為什麼想成為工程師 ( 請以自我期許帶入：前端、後端、或全端工程師 )

原本對電子書編輯的想像是做內容，但實際任職後，工作比較偏重在檔案編製上。因為電子書的技術有一部分是借用 HTML5 的規範，因此在 HTML 與 CSS 的編寫上小小有掌握。

除了編輯電子書，也會支援一些 app 的使用測試，以及擬定工作相關的系統需求，在與公司內部工程師溝通、工作的過程中，也經常幻想成為帥氣的工程師（？）

目前，我的工作比較像是一個找出問題、提出需求的角色，希望自己哪天也可以成為接受需求、解決問題的角色。

在查詢過一些資訊與實際的學習嘗試之後，希望自己第一個目標能成為前端工程師，並且最終成為後端工程師。

為了成為工程師，曾經做過什麼努力？

‧ 2019年1~6月假日　資策會「前端開發工程師精修班」

這個課程是偏於給在職人士上的，內容上比較是偏概述性質（課程大綱：https://www.iiiedu.org.tw/hcj/，不會有實作作業與相關檢討）

‧ 2019年12月~現在

上完資策會的精修班，講師有提醒說，這系列課程是帶我們認識前端工具是在做什麼，距離真正能就職還有一大段距離；課程結束後，我在學習過程大致遇到幾個問題與對應：

1. 不曉得怎麼應用學到的語言

以 js 為例，我上完資策會的課程後，甚至連 code war 上最基本的題目也解不出來，自己找的線上課程又有點太偏重觀念性

對應：找了家教，針對 js 做加強，主要訴求是從實際解題中，去熟習 js 的用法

2. 學習沒有系統性

在後續的自學中找不到方向，常常覺得自己東學一點、西學一點，有點雜亂無章

對應：在 medium 上看到 Huli 無私分享的程式導師實驗計畫第三期，參考這份去做為學習地圖

3. 缺乏敦促我持續達成階段性目標的來源

目前是在職中，常常下班後或假日就會不小心耍廢，少了一個有強度的、持續敦促自己往前的動力

對應：報名彭彭的課程，希望在規範時間內，確實要求自己完成階段性學習

對於工程師的職業想像是什麼？

戴著超厚黑框眼鏡坐在電腦前完成大事的人（誤）

如果是我自己的終極目標：後端工程師的話，守備範圍想像起來會較偏重於資料處理、資料庫管理與資訊安全，與前端工程師協作，合力完成網站的呈現

應該會是整項專案中強而有力的後盾

如果參與這個計畫，會怎麼安排學習時間？

參考照彭彭所說，每週約需投入40~60小時的話，周一到周五我應該會投入10~15小時（約2~3小時／天），週六日25~30小時（約12~15小時／天）；

並且在趕不上進度時，調用特休來輔助

如果參與這個計畫，預期會碰到什麼困難？你打算怎麼解決它？

最大的困難應該是自己的惰性與容易分心，檢視了自己的習慣與時間分配，預計會以下列方法將有限時間盡量極大化

1.利用一些輔助專心的工具，像是番茄鐘app

2.手機解鎖程式來輔助自己

3.試著刪除一兩個社群軟體
